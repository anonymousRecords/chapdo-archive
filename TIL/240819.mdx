## 📌 반복문의 동작 원리와 메모리 측면에서 바라본 반복문

대표적인 반복문 유형은 위와 같지만, 그 동작 원리는 유사하다.

1. 카운터와 같이 반복문에 사용될 변수를 초기화함.
2. 반복문의 조건을 확인함.
3. 조건문이 참이면 코드 블록을 실행함.
4. 코드 블록 실행 후 변수를 업데이트함.
5. 업데이트된 변수로 다시 조건을 확인하고 과정을 반복함.
6. 조건이 거짓이 되면 반복문을 종료함.

반복문을 메모리 관점에서 바라보면
- 변수는 메모리의 특정 위치에 저장됨.
- 반복문이 실행될 때마다 이 메모리 위치의 값을 읽고 업데이트함.
- 변수 값이 변경될 때마다 해당 메모리 위치의 내용이 갱신됨.

## 📌 구문과 메서드
구문과 메서드를 구분지어서 반복을 수행하는 기능을 살펴볼 수 있다.

먼저 구문과 메서드의 차이를 알아보면
- 구문(statement) : 언어의 기본 구조이며, 직접 사용할 수 있음
- 메서드(method) : 특정 객체에 속해 있으며, 해당 객체의 인스턴스나 프로토타입을 통해 호출해야 함

구문은 자체적인 실행 컨텍스트를 생성하지 않지만, 현재의 실행 컨텍스트(전역 실행 컨텍스트, 함수 실행 컨텍스트, eval 실행 컨텍스트...) 내에서 작동하며 때로는 자체적인 스코프를 가질 수 있다. 
반면 메서드는 호출될 때마다 새로운 실행 컨텍스트를 생성하고, 이 컨텍스트는 메서드가 속한 객체와 연관된다.

그래서 돌아오면
반복 구문(Loop Statements)으로는
- for loop
- while loop
- do-while loop
- for...of loop
- for...in loop

반복 관련 메서드로는
- forEach()
- map()
- filter()
- reduce()

가 있다고 할 수 있다.

## 📌 코드로 살펴본 반복 구문과 관련 메서드
while, do-while loop는 조건에 따라 반복을 제어해야 할 때 유용하다.
비유하자면 `사랑에 빠진 직진러`.
사랑하는 한 계속해서 같은 행동을 반복한다.
### while loop
```js
let i = 0;
while (i < 5) {
    console.log(i);
    i++;
}
```

### do-while loop
```js
let j = 0;
do {
    console.log(j);
    j++;
} while (j < 5);
```
---
그런데 for는 while, do-while이랑은 약간의 차이가 존재한다. 물론 while, do-while도 조건이 충족해서 사랑하겠지만, for는 더 세밀하다.
